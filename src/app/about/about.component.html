Data Binding in Angular

Data binding in Angular is a mechanism that allows you to synchronize data between the component and the view. There are four main types of data binding in Angular:

Interpolitation

Uses double curly braces {{ "{{ }}" }} to bind data from the component to the view.

This text is binded -> {{ title }}

Event Binding

Binds an event of an HTML element to a method in the component using parentheses ( )

Counter: {{ counter }}

Property Binding

Binds a property of an HTML element to a property in the component using square brackets [ ]

Two Way Binding

Combines property binding and event binding using the ngModel directive and the [( )] syntax (banana-in-a-box)

NGMODEL

ngModel is a directive in Angular that facilitates two-way data binding between form inputs and the component's properties. This means it synchronizes the data between the model (the component class) and the view (the template) automatically.

Hello, {{ name }}!

Directives in Angular

In Angular, directives are special markers on a DOM element (such as an attribute, element name, or CSS class) that tell Angular to attach a specific behavior to that element or even transform the element and its children. There are three main types of directives in Angular:

Component Directive

Component Directives: These are the most common type of directive and are essentially custom HTML elements. They encapsulate data, template, and behavior. Defined with the @Component decorator.

Structural Directives

These directives change the DOM layout by adding or removing elements. They typically use an asterisk (*) prefix. Examples include *ngIf and *ngFor

Services in Angular

In Angular, services are a way to encapsulate reusable functionality that can be shared across different parts of your application. They are typically used to centralize common tasks such as data fetching, logging, authentication, etc. Services promote modularity, maintainability, and testability in Angular applications.

Dependency Injection

Dependency Injection (DI) is a design pattern used in Angular to provide instances of objects (services, components, directives, etc.) that a class (the dependent) needs to perform its function. Instead of creating instances of dependencies within the dependent class, Angular's DI system injects these dependencies at runtime, making the code more modular, easier to maintain, and facilitating testing.

Decorators in Angular

In Angular, decorators are special functions that attach metadata to classes, methods, properties, or parameters. They play a crucial role in defining the structure and behavior of Angular applications.

Class Decorators:

These decorators are used to define classes and attach metadata to them. Common examples include @Component, @Directive, @Pipe, @Injectable, and @NgModule.

Example

{{ "@Component({ selector: 'app-hero', templateUrl: './hero.component.html', styleUrls: ['./hero.component.css'] })" }}

Property Decorators:

These decorators are used to attach metadata to properties within a class. Common examples include @Input, @Output, and @ViewChild.

Example

{{ "@Input() heroName: string" }}

{{ "@Output() notify = new EventEmitter()" }}

Method Decorators:

These decorators are used to add metadata to methods within a class. Example includes @HostListener.

Example

{{ " @HostListener('click', ['$event']) onClick(event: Event) { alert('Element clicked!');}" }}

Parameter Decorators:

These decorators are used to add metadata to parameters of a class constructor. Example includes @Inject.

Example

{{ "@Injectable() export class SomeService { constructor(@Inject(SomeDependency) private dependency) { } }" }}
